¿What are Semantic Tags?

Semantic HTML is the use of HTML markup to reinforce the semantics, or meaning, of information in web pages and web applications rather than merely defining their presentation or look. Semantic HTML is processed by traditional web browsers as well as by many other user agents. CSS is used to suggest its presentation to human users.

Well, Now. ¿What are the Semantic Tags?

Examples of non-semantic elements: "< div >", "< span >" - these do not provide information about the content of the page.
Examples of semantic elements: "< form >", "< table >" and "< img >" - clearly define its content.

¿Which are the ones used today?

Having a little bit in mind what semantic tags are, let's now see all the ones that are mainly used in web development.

Name Description
< section > The "section" tag helps us to define a section or a group of content of our web page.
< article > This tag helps us to build an article entry, such as a blog post.
< header > The header tag helps us define introductory content or navigable section such as a menu.
< footer > The footer is the bottom of the page, it generally contains important information, such as contact, legal sections, etc.
< aside > The aside tag helps us to place "less relevant" information (generally), it can also be used to place advertising.
< figure > This is used to place illustrations, images, etc.. This tag has its own flow, so it should not bother us with the rest of the flow of our web page.
< figcaption > Figcaption can add a text at the beginning or at the end of a figure element.
< details > This tag can function as a widget, storing the information without displaying it, unless the user opens it.
< main > This specifies the main content of the web page. Preferably it is expected that it does not contain things that have been used before.
< mark > The mark tag is used to highlight parts of the text.
< nav > Nav is used to be able to add "functionality", with this generally are made the menus with which we navigate within the page.
< summary > This is complemented by the "details" tag, which provides the button to show or hide the content.
< time > Provides a human- and computer-readable date. Can be interfaced with the user's calendar.
< img > It gives us the opportunity to place an image.
< form > With this tag we can create a form.
< table > As its name indicates, with this tag we can make a table, like the one I made to put this information.
